# План: пользовательский README

Цель: человек без опыта разработки открывает README, скачивает установщик под
свою ОС, проходит блокировку неподписанного приложения (на macOS — через
«Конфиденциальность и безопасность») и подключается к своему серверу за
5 минут, не заглядывая в dev-секции.

Сейчас README заточен под разработчика: «Быстрый старт» начинается с
`git clone`, секции «Скачать» нет вовсе, а совет про Gatekeeper устарел —
«правый клик → Open» на macOS 15+ для неподписанных приложений больше не
работает, единственный путь — Системные настройки → Конфиденциальность и
безопасность → «Открыть всё равно».

## Phase 1 — «Скачать и установить» (ядро)

1. **Шапка**: иконка, одна фраза о приложении, бейджи (Build status, Latest
   release, Downloads, платформы, лицензия). Бейджи оживут после публикации
   репозитория.
2. **Скриншот** Dashboard сразу под шапкой (placeholder до Phase 3).
3. **Секция «Скачать»**: таблица ОС → файл → прямая ссылка на Releases
   (стабильный релиз + `dev-latest` для смелых).
   - Имена файлов совпадают с версией релиза (проект поднят на 1.0.0, чтобы
     macOS-major ≥ 1 был честным, а не через маппинг `0.→1.`; см. Phase 3).
   - Требования: macOS (Apple Silicon; минимальную версию ОС уточнить при
     написании), Windows 10/11 x64, Debian/Ubuntu amd64.
4. **Установка на macOS** — ключевая секция (просьба пользователя):
   - Открыть DMG, перетащить `VLESS Client` в `Applications`.
   - Первый запуск: появится «Не удалось открыть, так как Apple не может
     проверить...» — это ожидаемо: приложение не подписано сертификатом
     Apple Developer. Нажать «Готово» (не «В корзину»).
   - Пошагово, с точными названиями пунктов на русском и английском:
     **Системные настройки → Конфиденциальность и безопасность →** прокрутить
     до раздела «Безопасность» → строка «Приложение "VLESS Client" было
     заблокировано» → кнопка **«Открыть всё равно» (Open Anyway)** →
     подтвердить паролем/Touch ID → в повторном диалоге «Открыть».
   - Скриншоты диалога блокировки и панели настроек (Phase 3).
   - Альтернатива для терминала:
     `xattr -d com.apple.quarantine "/Applications/VLESS Client.app"`.
   - Примечания: процедура повторяется после каждого обновления приложения
     (пока нет подписи); на macOS 13–14 достаточно правого клика → «Открыть».
5. **Установка на Windows**: SmartScreen → «Подробнее» → «Выполнить в любом
   случае»; MSI ставится per-user, админ-права не нужны.
6. **Установка на Linux**: `sudo apt install ./vless-client_*.deb`, запуск из
   меню приложений.

## Phase 2 — «Первое подключение» и переработка «Как пользоваться»

1. **Quick start после установки**: добавить сервер через Import Link
   (`vless://...`), нажать Connect, проверить, что IP сменился.
2. **Режимы**: короткая таблица «System Proxy vs TUN — какой выбрать», и что
   каждый режим просит по части прав на каждой ОС (перенести из текущей
   таблицы «Платформы», она перегружена).
3. Сжать существующие секции (сервера, подписки, маршрутизация, трей,
   хоткеи, настройки, обновление ядра через Settings → About) — содержание
   в порядке, нужна структура «частое — выше».
4. **Траблшутинг** дополнить: «приложение заблокировано/не удаётся открыть»
   → ссылка на секцию Privacy & Security; блокировка повторилась после
   обновления; на стоковом GNOME нет трея; TUN просит пароль — почему.
5. Убрать устаревшее: «правый клик → Open» как основной путь, `git clone`
   из начала README (уезжает в «Разработка» вниз).
